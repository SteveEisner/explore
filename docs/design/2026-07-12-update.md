# Design Update — 2026-07-12

This is evolving from a place where you dump docs and get explanations into an artifact-creation studio.

Instead of stopping at explaining, it helps you shape what the artifact should be, then builds it in place. The artifact is a shareable document that contains the app, and the .oui files are effectively the apps.

There’s also a source layer: a markdown file behind the document that captures the intent and origin. When you share the output, it’s a full HTML artifact, but it also has a view-source explanation of where it came from.

That means it’s one artifact, but richer than a typical finished output. Because it carries hints of its origins, you can ask it more questions than you could with a normal finished artifact.

One-hour plan: first, tighten the chat-and-tools editing loop. Make it easy for the system to see relevant documents, map screenshots to the right UI parts, and apply small edits quickly so iteration feels instant.

Second, add a view-source mode to the UI. The app should open as the interactive .oui experience, with a clear button to switch to a markdown source view using the existing markdown viewer, then switch back.
